# Netscape HTTP Cookie File — PLACEHOLDER ONLY
#
# DO NOT commit real browser cookies to this file or to git history.
# Real cookies are personal credentials: anyone with them can impersonate
# your logged-in browser sessions on the sites they cover.
#
# To supply real cookies to yt-dlp on Render:
#   1. Export a Netscape-format cookies.txt from a browser signed into YouTube.
#   2. In the Render dashboard, add it as a Secret File named `cookies.txt`.
#      Render mounts secret files at /etc/secrets/<filename>.
#   3. Ensure YTDLP_COOKIES_FILE=/etc/secrets/cookies.txt (set in render.yaml).
#   4. Redeploy. /api/health/ will report cookies.available: true.
#
# This committed file is intentionally empty of cookie rows so the repo stays
# safe to be public. `cookies.txt` is also gitignored in backend/.gitignore.
